# 브레맨 핵심 개념 정의

> 이 문서는 브레맨 플랫폼의 핵심 개념과 관계를 정의합니다.

---

## 1. 의뢰 · 제안 · 서비스 오퍼 관계

### 의뢰 (Request/Project)
클라이언트가 시작하는 프로젝트 요청.

```
클라이언트 → [프로젝트 생성] → 프로젝트 마켓 공개
```

**특징:**
- 클라이언트가 요구사항, 예산, 일정 정의
- 프로젝트 마켓에 노출
- 팀들이 제안을 제출

### 제안 (Proposal)
팀이 프로젝트에 응답하는 방식.

```
팀 → [제안 제출] → 프로젝트 응답
```

**특징:**
- 팀 리더가 작성
- 제안 예산/일정 제시
- 클라이언트가 수락/거절

### 서비스 오퍼 (Service Offer)
팀이 미리 제시하는 고정형 서비스.

```
팀 → [서비스 등록] → 서비스 마켓 노출
```

**특징:**
- 팀의 역량을 사전 정의
- 카테고리/예산/일정 범위 제시
- 클라이언트가 문의 가능

## 2. 정보 계층 구조

플랫폼 전반에 걸친 정보 우선순위:

```
1. 직무 (Role Type)     - 전문 영역 정의
2. 기술 (Skills)        - 객관적 역량 증명
3. 성향 (Animal Skin)   - 협업 스타일 참고
```

### 직무 유형 (Role Types)
- `frontend` - 프론트엔드 개발
- `backend` - 백엔드 개발
- `design` - UI/UX 디자인
- `pm` - 프로젝트 관리
- `devops` - 인프라/배포
- `mobile` - 모바일 개발
- `ai` - AI/ML 개발
- `data` - 데이터 분석

### 브레맨 성향 (Animal Skins)
협업 궁합 판단에만 사용, 업무 요구사항에 포함 안됨.

| 성향 | 특성 | 협업 스타일 |
|------|------|-------------|
| 🐴 말 | 리더십, 추진력 | 주도적, 목표지향 |
| 🐕 개 | 충성심, 협력 | 팀워크, 지원형 |
| 🐱 고양이 | 독립성, 전문성 | 자율적, 집중형 |
| 🐓 닭 | 소통, 창의성 | 아이디어, 표현력 |

## 3. 직무 레벨 계산

```
레벨 점수 = 기술 숙련도(60%) + 프로젝트 경험(40%)
          + 자격증 보너스 + 포트폴리오 보너스 + 팀 평가 보너스
```

### 레벨 등급
| 레벨 | 점수 범위 | 설명 |
|------|-----------|------|
| Lv.5 | 90-100 | 전문가 (Expert) |
| Lv.4 | 70-89 | 숙련자 (Advanced) |
| Lv.3 | 50-69 | 중급자 (Intermediate) |
| Lv.2 | 30-49 | 초급자 (Beginner) |
| Lv.1 | 0-29 | 입문자 (Novice) |

## 4. 추천 시스템

단일 로직 기반 추천:

```
총점 = 기술 매칭(60%) + 레벨 요구(20%) + 성향 궁합(20%)
```

**노출 위치:**
- `/dashboard` - 추천 팀
- `/teams/:id` - 지원 시 슬롯 적합도
- `/projects/:id` - 클라이언트 팀 추천

---

## 5. 워크플로우

### 클라이언트 관점
```
의뢰 등록 → 제안 수신 → 팀 선택 → 계약 체결 → 마일스톤 관리 → 완료/리뷰
```

### 팀 관점
```
프로젝트 탐색 → 제안 제출 → 계약 체결 → 마일스톤 수행 → 완료/리뷰
```

### 개인 관점
```
프로필 설정 → 팀 탐색 → 지원/수락 → 팀 활동 → 프로젝트 참여
```
